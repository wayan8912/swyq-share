上门服务O2O平台定位
本平台是蒙古国乌兰巴托的上门服务信息平台。用户在平台选定服务与个体商户后下单，然后由个体商户本人提供服务。平台提供信息展示、订单管理、支付接入、人工审核、投诉处理、退款协助与财务记录能力。
⚙ 可配置或优化规则
具体规则，最后由客户与蒙古国法务、税务、监管方确认后配置。
开发方责任边界
开发方负责将当地法务、监管方与客户确认后的要求，落实为系统配置、审核流程、下单规则、日志记录与后台管理能力。

6.1　第一版最短链路

支付与清算
◇ 待确认

决定了支付接入、对账逻辑与数据库设计，并联动第 08 节税务。

7.1　清算方案对照

选项资金路径平台收入风险建议
选项1用户在线支付，本地持牌支付机构完成收款、分账、退款中介费 + 平台服务费平台不长期沉淀资金，风险较低，需要对接本地支付机构推荐
选项2用户付款进入平台账户，平台代收再向个体商户结算扣除中介费 + 平台服务费资金沉淀、二次清算、备付金、审计不默认
风险提醒：如果平台代收后再结算，就不能再简单按信息中介平台处理，需要单独做支付合规、二次清算、资金隔离、退款备付、对账审计、反洗钱、税务合规与结算规则设计。

7.2　第一版财务记录字段

记录项是否需要用途
订单总金额需要付款与退款依据。
平台中介费需要平台收入依据。
平台服务费需要平台收入依据。
个体商户应收需要结算依据。
支付状态需要确认是否付款成功。
退款状态需要处理退款与争议。
结算状态需要确认是否已向商户结算。
支付回调原始记录需要排查重复回调、失败回调与支付争议。
08

税务与凭证
◇ 待确认

第一版建议：信息中介平台采用「各开各的、各负其责」。个体商户对自己的服务收入负责，平台对自己的中介费与平台服务费负责，平台不为个体商户代缴税款。
选项含义风险建议
各开各的个体商户处理服务收入凭证，平台处理平台费用凭证个体商户可能不处理凭证第一版推荐
平台技术协助开具平台接入电子税票接口，协助商户完成技术操作需授权、失败重试、退款冲正，可能加重平台责任第二版谨慎
平台统一向用户开具平台对用户开服务凭证平台可能被理解为销售主体不建议
平台代缴税款平台替商户处理税款平台责任过重不建议

个体商户审核
✓ 已确认● 必做

9.1　基础审核材料

审核项第一版要求用途
真实姓名必须提交确认服务主体。
蒙古国手机号必须验证登录、通知、追责。
身份资料必须提交并人工审核防止虚假入驻。
展示照片建议提交方便用户确认上门人。
从业 / 资质材料按服务类目配置正规按摩等高关注类目可要求额外材料。
收款账户必须提交用于结算。
税务声明必须提交明确商户自行负责税务。
服务区域必须设置限制可接单范围。
服务类目必须审核决定是否允许上线。

9.2　本人到场规则

第一版强制：必须个体商户本人到场。不得冒名、不得转单、不得结伴代替服务。
禁止事项风险处理建议
冒名上门用户安全与责任无法确认一经确认，限制接单或冻结个体商户账号。
转单给他人平台人工审核失效禁止。
结伴用户预期与现场安全风险上升禁止，除非监管另行允许。
使用他人收款账户结算主体不清个体商户账号标记异常并人工审核。

10.1　可配置项

配置项可配置内容用途
类目是否开启开启 / 关闭 / 仅指定商户开放监管确认前后灵活调整。

订单流程用户选服务与商户后下单，商户接单或拒单，金额锁定。
支付记录支付发起、状态、退款状态、支付回调。
财务记录订单金额、平台服务费、商户应收、结算状态。
投诉与退款用户申请、商户说明、运营端处理。
日志订单状态、审核、后台操作、支付回调日志。

14.2　暂不开发

暂不开发内容原因
多人报价不符合当前最简单订单模式。
平台自动派单增加责任风险。
平台钱包支付与资金沉淀风险高。
平台统一开税务凭证改变平台税务主体边界。
平台代缴商户税款平台责任过重。
微信小程序 / 原生手机应用第一版先做网页版。

风险矩阵

15.1　高风险

风险触发条件影响处理建议
平台沉淀资金平台代收后再结算支付、清算、备付金、反洗钱、审计风险上升优先本地支付机构分账。
税务主体改变平台统一向用户开凭证平台可能被理解为销售主体第一版各开各的。
服务场景错配未按监管 / 法务要求配置地点、时间、资料影响合规与运营安全高关注类目必须支持后台配置与审核。
冒名 / 转单接单人不是上门人身份审核失效，责任不清本人到场，禁冒名、转单、结伴。
用户位置泄露接单前展示完整地址隐私与安全风险接单后才展示完整地址。

15.2　中风险

风险触发条件处理建议
价格纠纷现场加价或服务范围不清下单金额锁定，变价须用户确认。
私下交易用户与商户绕开平台平台内沟通留痕，违规处罚。
凭证争议用户要凭证，个体商户无法提供展示个体商户凭证能力，平台只处理自身费用凭证。
频繁拒单个体商户接单不稳定记录拒单率，影响展示排序。
服务质量纠纷高关注类目资质或健康证审核不严强制资质审核、过程留痕、投诉协调。
关键条款误译英文 / 中文 / 西里尔蒙古文含义不一致关键条款必须人工审核。

15.3　低风险

事项处理
分类 / 类目 / 城市扩展预留字段，灰度上线。
界面与价格展示优化按反馈调整。
新增语言在既有多语言框架内扩展。
